## 概要

GitHub Actions 上で OpenReview API に対してリクエストを行うと `403 Forbidden` が返るようになった。ブラウザからは同エンドポイントへのアクセスが可能であるため、GitHub Actions の IP レンジが OpenReview 側でブロックされていると判断した。

対策として OpenReview API v2 の JWT 認証を導入し、認証済みリクエストとして論文データを取得できるようにする。

## やること

- [ ] `internal/config/config.go` に `OpenReviewEmail`, `OpenReviewPassword` フィールドを追加する
- [ ] `internal/openreview/openreview.go` の `Client` 構造体に `token` フィールドを追加する
- [ ] `openreview.Client` に `Login(email, password string) error` メソッドを追加する
- [ ] `GetNotes()` に `Authorization: Bearer <token>` ヘッダーの付与ロジックを追加する
- [ ] `cmd/dailybot/main.go` に認証ブロックを追加する
- [ ] `.github/workflows/daily.yml` に `OR_EMAIL`, `OR_PASSWORD` の環境変数を追加する
- [ ] `Login()` のユニットテストを追加する（正常系・異常系）
- [ ] `GetNotes()` のヘッダー付与テストを追加する

## 方針

- `openreview.Client` に `Login()` メソッドを追加し、取得したトークンをフィールドで保持する
- `GetNotes()` はトークンが空でない場合にのみ `Authorization` ヘッダーを付与する（後方互換を維持）
- 認証情報（`OR_EMAIL`, `OR_PASSWORD`）は GitHub Secrets で管理する
- 両方セットされている場合のみ認証を試みる。どちらかが空の場合は認証をスキップする（ローカル開発・DRY_RUN との互換性維持）
- 認証失敗時は即 FATAL で終了する（リトライ・フォールバックなし）
- トークンのキャッシュ・永続化は行わない（毎回ログインする方式）

## 方法

### 1. Config の変更（`internal/config/config.go`）

```go
type Config struct {
    // ... 既存フィールド ...
    OpenReviewEmail    string
    OpenReviewPassword string
}
```

環境変数 `OR_EMAIL`, `OR_PASSWORD` を読み込む。どちらも任意項目とする。

### 2. Client 構造体の変更（`internal/openreview/openreview.go`）

```go
type Client struct {
    httpClient *http.Client
    BaseURL    string
    UserAgent  string
    token      string  // 追加: 空文字 = 未認証
}
```

### 3. Login() メソッドの追加

```
POST https://api2.openreview.net/login
Content-Type: application/json

{"id": "<email>", "password": "<password>"}
```

レスポンス形式:
```json
{"token": "<JWT>", "user": {...}}
```

- `token` フィールドを取得し `c.token` に保存する
- ステータスコードが 200 以外の場合はエラーを返す

### 4. GetNotes() の変更

```go
if c.token != "" {
    req.Header.Set("Authorization", "Bearer "+c.token)
}
```

### 5. main.go の変更

```go
orClient := openreview.NewClient(cfg.CustomUserAgent)

if cfg.OpenReviewEmail != "" && cfg.OpenReviewPassword != "" {
    if err := orClient.Login(cfg.OpenReviewEmail, cfg.OpenReviewPassword); err != nil {
        log.Fatalf("FATAL: failed to login to openreview: %v", err)
    }
    log.Println("INFO: Authenticated to OpenReview.")
}
```

### 6. daily.yml の変更

```yaml
OR_EMAIL: ${{ secrets.OR_EMAIL }}
OR_PASSWORD: ${{ secrets.OR_PASSWORD }}
```

## 事前準備（ユーザー作業）

1. [https://openreview.net](https://openreview.net) でアカウントを作成する
2. GitHub リポジトリの **Settings > Secrets and variables > Actions** で以下を登録する
   - `OR_EMAIL`: 登録したメールアドレス
   - `OR_PASSWORD`: 登録したパスワード
