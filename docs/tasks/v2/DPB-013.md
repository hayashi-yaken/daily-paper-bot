## 概要

「一度投稿された論文は再投稿しない」重複防止機構を完全に廃止する。`internal/storage` パッケージ、`data/posted.json` ファイル、GitHub Actions の auto-commit ステップを含む関連コード／データ／ドキュメントを一式撤去し、Bot が実行のたびに OpenReview から取得した有効候補からランダムに 1 本を選定する運用に切り替える（再投稿が許容される運用）。

## 方針

- 「最小限の無効化」ではなく「機構ごとの完全削除」を選ぶ。`storage` パッケージを残すと「何のためにあるか不明」な死蔵コードになるため。
- `docs/PROJECT_SPEC.md` の旧 MVP 記述（セクション 10 など）は履歴として残し、冒頭バナーで廃止状態を明示する（v1 task ticket と同様の履歴保存方針）。
- GitHub Actions の権限は最小権限 (read-only) に縮退させる。`posted.json` 自動コミットが不要になったため `contents: write` も不要。
- `data/posted.json` は履歴情報としても保持しない（再投稿許容運用に振り切るため、過去履歴を参照する用途がない）。
