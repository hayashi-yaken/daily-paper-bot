## 概要

Go言語によるバッチアプリケーションの基本的なプロジェクト構造をセットアップします。仕様書で定義されたリポジトリ構成に基づき、ディレクトリと初期ファイルを配置します。

## やること

- [x] Goモジュールの初期化 (`go mod init`)
- [x] 仕様書に基づいたディレクトリ構造の作成
  - `cmd/dailybot/`
  - `internal/`
  - `data/`
  - `.github/workflows/`
- [x] エントリーポイントとなる `cmd/dailybot/main.go` の作成
- [x] `.gitignore` ファイルの作成（Goの一般的な設定）
- [x] `README.md` の作成（プロジェクト概要を記述）

## 方針

- 仕様書「6.1 リポジトリ構成（例）」に厳密に従う。
- `internal` パッケージを活用し、外部から直接利用されないコンポーネントを配置する。
- `main.go` は、現時点では "Hello, World!" を出力する程度の最小限の実装とする。

## 方法

### 1. Goモジュール初期化

```bash
go mod init github.com/hayashi-yaken/daily-paper-bot
```

### 2. ディレクトリ作成

```bash
mkdir -p cmd/dailybot internal data .github/workflows
```

### 3. ファイル作成

```bash
touch cmd/dailybot/main.go .gitignore README.md data/posted.json
```

`data/posted.json` は初期状態（空のJSONオブジェクト）で作成します。

```json
{
  "posted": {}
}
```
